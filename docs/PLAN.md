# 이직추천 서비스 — 프로젝트 계획서 v2

> 단순 AI 추천 도구가 아니라, **이직자 내부자 데이터 플랫폼**
> "내 정보를 주면 → 내가 얻을 수 없었던 정보를 받는다"

---

## 1. 서비스 정체성 재정의

### 진짜 하고 싶은 것

잡플래닛, 블라인드에도 없는 데이터가 있다.  
**이직을 진지하게 고민하는 사람이 자기 입으로 꺼내는 내부 정보.**

- 내 회사 실제 연봉 테이블 (공개된 공시 아님, 실제로 내가 받는 숫자)
- 왜 떠나려 하는지 (표면적 이유가 아닌 진짜 이유)
- 지금 회사의 진짜 장단점 (입사 후에야 아는 것들)
- 경영진/리더십에 대한 내부 신뢰도
- "1년 후에도 여기 있을 것 같나요?" — 이탈 의향 지표

이 데이터는 **자발적 교환** 없이는 절대 수집 불가능하다.  
그 교환의 대가가 바로 "맞춤 이직 추천"이다.

### 포지셔닝 차별화

| | 잡플래닛 | 블라인드 | 이 서비스 |
|---|---------|---------|----------|
| 핵심 데이터 | 재직자 리뷰 | 익명 커뮤니티 | 이직 의향자 내부 정보 |
| 수집 방식 | 리뷰 작성 후 열람 | 인증 후 자유 게시 | 이직 추천 교환 |
| 데이터 품질 | 감정적, 구조화 낮음 | 신뢰도 낮음 | 구조화, 이직 의향 명확 |
| 핵심 가치 | 후기 열람 | 익명 대화 | 개인화 추천 + 데이터 자산 |

## 2. 가치 교환 구조 (핵심 메커니즘)

```
사용자가 제공하는 것         사용자가 받는 것
─────────────────────       ──────────────────────
현황 입력 (쉽게, 3분)   →   맛보기 추천 결과 (잠금 해제 유도)
         ↓
회사 내부 정보 입력      →   전체 AI 이직 추천 결과 잠금 해제
(진짜 연봉, 퇴사 이유,        (추천 기업군 + 갭 분석 + 액션플랜)
 회사 장단점, 이탈 의향)
         ↓
이메일 등록             →   맞춤 채용공고 알림 수신
```

### 왜 이 구조가 작동하는가

- 이직을 고민하는 사람은 **이미 회사 정보를 털어놓을 준비가 되어 있다**
- "내가 왜 떠나고 싶은지"를 물어보는 것 자체가 공감 → 신뢰 → 참여 유도
- 알림 신청은 이직 의향이 있는 사람에게 가장 자연스러운 다음 행동

## 3. 수집 데이터 설계

### 3.1 현황 입력 (Stage 1 — 쉽게, 필수)

| 데이터 | 형식 | 비고 |
|--------|------|------|
| 현재 직무 | 버튼 선택 | 개발/기획/마케팅/영업/디자인/금융/HR/기타 |
| 현재 업종 | 드롭다운 | IT/금융/제조/유통/의료/교육/컨설팅 등 |
| 경력 연수 | 버튼 선택 | 1년미만/1-3년/3-5년/5-7년/7-10년/10년이상 |
| 현재 연봉 | 버튼 선택 | 2000~3000/3000~4000/.../1억이상 |
| 주요 스킬 | 태그 입력 | 자유 입력 |
| 현재 회사 규모 | 버튼 선택 | 스타트업/중소/중견/대기업/외국계/공기업 |

### 3.2 회사 인사이트 입력 (Stage 2 — 핵심 가치 교환)

| 데이터 | 형식 | 왜 가치 있는가 |
|--------|------|----------------|
| **실제 연봉 (정확한 숫자)** | 숫자 입력 | 공시되지 않은 실제 연봉 데이터베이스 |
| 현재 회사명 | 텍스트 (선택) | 회사별 연봉/이탈률 집계 |
| 직급/레벨 | 버튼 선택 | 직급별 연봉 밴드 구성 |
| 퇴사/이직 고민 이유 | 구조화 복수선택 | 회사별 주요 이탈 원인 |
| 회사 장점 top3 | 구조화 복수선택 | 회사별 강점 집계 |
| 회사 단점 top3 | 구조화 복수선택 | 회사별 약점 집계 |
| 경영진 신뢰도 | 1~5점 | 리더십 지표 |
| 성장 가능성 체감 | 1~5점 | 미래 이탈 예측 |
| 이탈 의향 | "1년 후 여기 있을 것 같나요?" 0~100% | 이탈 위험도 지표 |
| 추천 의향 (NPS) | 0~10 | 회사 건강도 종합 지표 |

### 3.3 알림 신청 (Stage 3 — 선택)

| 데이터 | 형식 | 활용 |
|--------|------|------|
| 이메일 | 입력 | 맞춤 채용공고 알림 발송 |
| 이직 타임라인 | 버튼 선택 | 알림 긴급도 조절 |
| 선호 회사 규모 | 복수선택 | 알림 필터링 |

## 4. 화면 플로우 (재설계)

```
랜딩 페이지
  "당신의 이직, AI가 먼저 찾아드립니다"
  [지금 분석 시작하기]
       │
       ▼
[Stage 1] 현황 입력 (쉽게, 2분)
  직무 / 업종 / 경력 / 연봉구간 / 스킬 / 회사규모
       │
       ▼
[맛보기 잠금 화면]
  "분석이 완료됐어요!"
  ━━━━━━━━━━━━━━━━━━━━
  ▓▓▓▓▓▓▓▓▓▓▓ (블러 처리된 추천 미리보기)
  "🔒 더 정확한 추천을 위해 회사 정보를 알려주세요"
  "알려주신 정보는 익명으로 집계되어 서비스 개선에 사용됩니다"
  [회사 정보 입력하고 전체 결과 보기 →]
       │
       ▼
[Stage 2] 회사 인사이트 입력 (2분)
  실제 연봉 / 퇴사 이유 / 회사 장단점 / 경영진 / NPS
       │
       ▼
[AI 분석 중] 로딩 + 스트리밍
       │
       ▼
[전체 결과] 잠금 해제 ✨
  진단 요약 / 추천 기업군 / 갭 분석 / 액션플랜
       │
       ▼
[Stage 3] 알림 신청 (선택)
  "맞춤 채용공고가 올라오면 알려드릴까요?"
  [이메일 입력하고 알림 받기]
```

## 5. 집계 데이터 활용 (서비스 고도화 시)

충분한 데이터가 쌓이면 사용자에게도 피드백으로 돌려줄 수 있다:

- "당신 회사(또는 같은 규모/업종)에서 이직한 사람들이 가장 많이 간 곳"
- "비슷한 직무/연차의 평균 연봉 vs 당신의 연봉"
- "당신 회사의 이탈 위험도 지수 (익명 집계)"
- "이 직군에서 요즘 가장 핫한 이직처 TOP5"

이 기능들이 생기면 **기존 사용자의 재방문 이유**가 된다.

## 6. 비즈니스 모델

### Phase 1 — 데이터 자산 구축 (무료)
- 사용자에게 무료로 추천 제공
- 내부자 데이터 수집 + 축적

### Phase 2 — 데이터 기반 수익화
| 고객 | 상품 | 가치 |
|------|------|------|
| **기업 HR팀** | "우리 회사 직원들의 이탈 위험도 리포트" | 핵심 인재 이탈 예측 |
| **경쟁사 HR** | "경쟁사 출신 이직자 동향" | 채용 타겟팅 |
| **헤드헌터** | "특정 회사 출신 인재 풀 접근" | 리크루팅 가속 |
| **VC/미디어** | "기업 건강도 지수 리포트" | 투자/기사 인사이트 |

### Phase 3 — 채용 연결 수익화
- 추천 기업 채용공고 클릭 → 이직 성공 시 수수료 (헤드헌팅 모델)
- 기업 채용 광고 (이직 의향자에게 타겟팅)

## 7. 기술 스택

```
Frontend    Next.js 14+ (App Router) + TypeScript
Styling     Tailwind CSS + shadcn/ui
AI Engine   Anthropic Claude API (claude-sonnet-4-6)
Database    Supabase (회사 인사이트 + 이메일 저장)
Email       Resend (알림 이메일 발송)
Deployment  Vercel
```

### Supabase 테이블 설계 (초안)

```sql
-- 회사 인사이트 (Stage 2 데이터)
company_insights (
  id, created_at,
  job_category, industry, experience_years,
  salary_range,           -- Stage 1 연봉 구간
  actual_salary,          -- Stage 2 실제 연봉 (핵심)
  company_name,           -- nullable (선택 입력)
  company_size, job_level,
  resignation_reasons,    -- jsonb array
  pros, cons,             -- jsonb array
  mgmt_trust_score,       -- 1~5
  growth_score,           -- 1~5
  stay_probability,       -- 0~100
  nps_score               -- 0~10
)

-- 알림 신청
alert_subscriptions (
  id, email, created_at,
  timeline, preferred_sizes,  -- jsonb
  job_category, industry
)
```

## 9. 주요 고려사항

| 이슈 | 대응방안 |
|------|---------|
| 개인정보 | 회사명 선택, 연봉은 집계만, 개인 식별 불가 구조 |
| 데이터 신뢰성 | 이직 의향자 = 실제 경험자, 구조화 질문으로 일관성 확보 |
| 초기 콜드스타트 | AI 추천은 집계 데이터 없어도 작동 (Claude 활용), 데이터는 천천히 쌓임 |
| 회사 입력 거부 | Stage 2 건너뛰기 옵션 제공 (단, 결과 잠금 유지) |
| Claude API 비용 | Stage 2 완료 후에만 API 호출 (입력 포기 시 과금 없음) |
