Problem

이직 시장의 가장 큰 문제는
정보의 비대칭입니다

🕵️

진짜 정보가 없다

실연봉, 퇴사 이유, 조직문화 같은 핵심 정보는 외부에 공개되지 않습니다.

🎲

감으로 결정한다

잡플래닛 별점·지인 한두 명의 말에 의존해 인생의 큰 결정을 내립니다.

📉

후회로 이어진다

이직자의 상당수가 "들은 것과 다르다"며 1년 내 재이직을 고민합니다.

Key Insight

그래서 우리는 추천만 하지 않고,
데이터를 만듭니다

사용자가 자기 회사의 내부 정보(실연봉·퇴사이유·장단점·NPS)를 입력해야 추천·알림을 받을 수 있는 가치 교환(value exchange) 구조.

현황 입력 (쉬움) → 🔒 결과 잠금 → 내부자 정보 제공 → 🔓 추천 해제 + 데이터 축적

쓸수록 데이터가 쌓이고, 데이터가 쌓일수록 추천이 정확해지는 선순환.

명함첩 연동 — 외부 데이터로 확장

서베이뿐 아니라 리멤버 같은 명함첩을 연동해 추천 근거를 두 배로.

자산①

🕸️ 인맥 네트워크 맵

내 명함첩 = 내가 네트워킹하는 회사들의 지도. 인맥이 닿아있는 회사를 찾아 레퍼럴(지인 추천) 입사 가능성이 높은 곳을 추천.

자산②

🌿 유사 커리어 경로

사람들이 이직할 때마다 명함이 바뀜 = 개인의 이직 이력. 나와 비슷한 커리어를 탄 사람들이 다음에 간 회사를 분석해 추천.

이직 추천 서비스
Pipeline

명함 한 장이 추천이 되기까지

📇 명함첩 연동 → namecards 적재 → company_network_edges
(회사 동시등장 = 네트워크)
동일인 회사 변경 감지 → contact_transitions
(명함 교체 = 이직) → career_paths
(커리어 궤적 누적)

recommend_by_career_path() — 내 회사 시퀀스와 겹치는 경로를 가진 사람들이 "다음에 간 회사"를 빈도·유사도순으로 반환하는 PostgreSQL 함수. 이름·연락처는 저장하지 않고 해시로만 처리.

이직 추천 서비스
Recommendation Engine

3개의 축을 결합한 추천 점수

축 1

서베이 적합도

직군·경력·스킬·연봉 매칭

축 2

네트워크 근접도

명함첩 인맥 밀도

축 3

커리어 경로 유사도

유사 궤적 peer의 선택

적합도 × 0.5+ 네트워크 × 0.25+ 경로 × 0.25= 최종 추천 랭킹

가중치는 튜닝 가능 · 결과는 recommendation_scores 테이블에 영속화(재현·AB·튜닝)
