# 이직추천 서비스 — UI/UX 설계 문서 v2

> 핵심 메커니즘: 현황 입력 → 잠금 화면 → 회사 정보 입력 → 전체 결과 잠금 해제 → 알림 신청
> 작성일: 2026-05-29

---

## 2. 전체 화면 구성

| # | 화면 | 설명 |
|---|------|------|
| 1 | 랜딩 | 서비스 소개, CTA |
| 2 | Stage 1 — 현황 입력 | 직무/연봉/스킬 (쉽게, 2분) |
| 3 | 맛보기 잠금 화면 | 결과 미리보기 + Stage 2 유도 |
| 4 | Stage 2 — 회사 인사이트 입력 | 핵심 데이터 교환 |

| 7 | 알림 신청 | 이메일 + 완료 |

### 3.2 Stage 1 — 현황 입력

**목표: 쉽고 빠르게. 선택형 위주, 타이핑 최소화**

```
┌──────────────────────────────────────┐
│ ←    현황 입력          1 / 2        │
│ ████████████░░░░░░░░░░  50%          │  ← Progress Bar
├──────────────────────────────────────┤
│                                      │
│  지금 어떤 일 하세요?                 │
│                                      │
│  [개발·엔지니어]  [기획·PM]           │
│  [마케팅·광고]    [영업·BD]           │
│  [디자인·UX]      [금융·회계]         │
│  [HR·총무]        [제조·생산]         │
│  [기타 ▾]                            │  ← 버튼 그리드 (2열)
│                                      │
│  ────────────────────────────────   │
│                                      │
│  경력은요?                            │
│  [1년미만] [1~3년] [3~5년]           │
│  [5~7년]   [7~10년] [10년+]          │  ← 버튼 그룹
│                                      │
│  ────────────────────────────────   │
│                                      │
│  현재 연봉 구간                       │
│  [~3천] [3~4천] [4~5천] [5~6천]     │
│  [6~8천] [8천~1억] [1억+]            │  ← 버튼 그룹 (wrap)
│                                      │
│  ────────────────────────────────   │
│                                      │
│  주요 스킬이나 역량 (최대 5개)        │
│  ┌──────────────────────────────┐   │
│  │ React ×  Python ×  기획 ×   │   │
│  │ [입력하고 Enter...]          │   │  ← 태그 인풋
│  └──────────────────────────────┘   │
│  예: React, 영업관리, SQL, B2B협상   │  ← 힌트
│                                      │
│  ────────────────────────────────   │
│                                      │
│  현재 회사 규모                       │
│  [스타트업] [중소기업] [중견기업]     │
│  [대기업]   [외국계]   [공기업]       │
│                                      │
├──────────────────────────────────────┤
│           [다음 단계 →]              │  ← Fixed Bottom
└──────────────────────────────────────┘
```

**UX 포인트**
- 업종 선택란 제거 → 직무+회사규모로 충분히 추론 가능, 폼 단축
- 스킬 입력: 인기 태그 추천 (직무 선택 시 자동 제안)
- 모든 항목 선택 전까지 "다음" 버튼 비활성 (회색), 선택 완료 시 Indigo 활성

**입력 상태 타입**
```typescript
interface Stage1Data {
  jobCategory: string        // '개발·엔지니어' | '기획·PM' | ...
  experienceYears: string    // '1~3년' | '3~5년' | ...
  salaryRange: string        // '3~4천' | '4~5천' | ...
  skills: string[]           // 최대 5개
  companySize: string        // '스타트업' | '대기업' | ...
}
```

- 단, 결과 하단에 "정확도를 높이려면 회사 정보를 추가해보세요" 배너 유지

**UX 포인트**
- 회사명: 선택 사항임을 명확히, 입력 시 "더 정확해져요" 인센티브 메시지
- 실제 연봉 인풋: 숫자 키보드 자동 호출 (inputMode="numeric")
- 이탈 의향 슬라이더: 좌우 레이블로 극단값 안내
- NPS 10개 버튼: 번호 선택 시 해당 색상으로 강조 (0~6 회색, 7~8 노랑, 9~10 초록)
- 전체 항목 미입력 가능 → 최소 "이직 이유 1개 이상 선택"만 강제

```

                        │
                        └─ [건너뛰기] ─→ 기본 결과 (/result?basic=true)
```

## 5. URL & 상태 전략

- 폼 상태: `sessionStorage` (새로고침 시 보존, 탭 닫으면 삭제 — 개인정보 보호)
- 결과 공유: `/result?d=BASE64_ENCODED_PARAMS` (연봉 실수치 제외, 구간만 인코딩)
- 알림 신청 완료: `/result?alert=success` → 완료 토스트
